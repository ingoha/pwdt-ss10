Engine Output Configurator

Der Engine Output Configurator definiert die Art und den Umfang der Einträge in den Logfiles.

Die sitebezogenen Konfigurationsdateien des Engine Output Configurators werden im Verzeichnis $HCISITEDIR/eoalias gespeichert.

 

 

Die Konfiguration erfolgt in 2 Schritten:

·       Erstellung von Aliase, mit denen die Art und der Umfang der Einträge definiert werden.

·       Zuordnung der Aliase zu einem Prozess oder einen bestimmten Thread im Network Configurator.

 

Wird keine explizite Konfiguration eingerichtet, werden dennoch Warnungen und Fehlermeldungen in die Logfiles geschrieben.

Erstellung eines Alias

Ein EO Alias kann mehre Einträge enthalten.

Jeder Eintrag besteht aus 5 Angaben.

 

 

Komponente

Ausprägungen

Action

Enable           Aktivierung der Reportingfunktion der hier angegebenen Module bzw. Submodule.

Disable          Deaktivierung der Reportingfunktion der hier angegebenen Module bzw. Submodule.

Module

Die Auswahl der Module (siehe unten) erfolgt über den Button Edit…

Submodule

In Abhängigkeit der gewählten Module können Submodule angegeben werden.

Die Auswahl der Submodule erfolgt über den Button Edit…

Severity

Info                  Die Engine Output Informationen sind anwenderorientiert.

Debug            Die Engine Output Informationen sind entwicklungsorientiert. (Dieser Modus wird i. d. R. vom Support angefordert).

Level

Ausgabe von minimal Informationen.

Ausgabe von wenigen Informationen.

Ausgabe von mehr Informationen.

Ausgabe von vielen Informationen.

*                      Ausgabe aller Detailinformationen.

 

 

Sind die Einträge erstellt, wird der Alias gespeichert. Hier kann unterschieden werden, auf welcher Ebene, Root oder Site, die Speicherung erfolgen soll.
