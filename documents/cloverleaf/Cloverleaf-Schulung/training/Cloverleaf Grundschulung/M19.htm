UPoCs und TCL / Java

UPoCs (User Point of Control)

 

CLOVERLEAF® bietet die Möglichkeit an, zusätzlich zu den Konfigurationsbausteinen, die mit den graphischen Tools erstellt werden, mittels Programmierung u. a. direkt Einfluss auf die Daten innerhalb einer Nachricht zu nehmen.

 

Die Programmierung kann entweder in TCL oder in Java erfolgen.

TCL Tool Command Language oder Java

 

Mit dem Script Editor bietet CLOVERLEAF® die Möglichkeit TCL Prozeduren zu erstellen. Der Level 2 Kurs beschäftigt sich ausschließlich mit dem Thema Erstellung und Einbindung von TCL Prozeduren. Sämtliche Konfigurationsdateien des Script Editors werden im Verzeichnis $HCISITEDIR/tclprocs gespeichert.

 

CLOVERLEAF® unterscheidet die 3 folgenden Typen von TCL

·       Transaction identification determination code

·       Code fragments

·       Tcl Procedure Stream procedures

 

Transaction identification determination codes kommen im Bereich der Trx Id determination (Reiter Inbound) zum Einsatz. D. h. anstelle der Eingabe von Offset und Length kann auch eine TCL Prozedure oder Java Klasse die Trx Id bestimmen.

 

Code fragments werden in den Xlates, im xlate-Action-Preprocessor bzw. im xlate-Action-Prostprocessor, eingesetzt.

 

Tcl Procedure Stream procedures (TPS) haben i. d. R. im Vergleich zu den beiden anderen Typen vollen Zugriff auf die gesamte Nachricht. Typische Einsatzgebiete sind hier die TPS Inbound Data, zum Aufsplitten von großen Dateien mit mehreren Datensätzen oder die TPS Outbound Data, zum Erstellen eines Dateinamens.

 

Anstelle von TCL Prozeduren können mittlerweile in fast allen UPoCs auch Java Klassen zum Einsatz kommen.
