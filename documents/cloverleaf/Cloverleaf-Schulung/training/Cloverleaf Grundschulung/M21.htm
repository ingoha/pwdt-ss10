Alert Configurator

Zur Unterstützung und Automatisierung der administrativen Arbeiten bietet CLOVERLEAF® die Möglichkeit mit dem Alert Configurator Alarme einzurichten.

Die Konfigurationsdateien des Alert Configurators werden im Verzeichnis $HCISITEDIR/Alerts gespeichert.

Funktionsweise

Mit Initialisierung einer neuen Site stellt CLOVERLEAF® systembezogene Alarme in der default.alrt zur Verfügung. Diese Datei kann um kundenbezogene Alarme erweitert werden. Die default.alrt Datei ist die CLOVERLEAF® Standard-Alarm-Datei.

 

Das Speichern kundenbezogener Alarme in eine andere Datei ist auch möglich. Sollten eine oder mehrere andere Dateien die benötigten Alarme enthalten, müssen diese Dateien mit dem Site Daemon aktiviert werden (siehe Site Daemons).

 

Um einen neuen Alarm einzurichten wird entweder eine neue Datei angelegt oder eine bestehende Datei geöffnet. Mit dem Befehl New Alert wird das Eigenschaftsfenster des Alarms geöffnet. Hier wird der Trigger (Auslöser) und die Aktion (was soll dann geschehen) eingerichtet.

 

 

 

 

Trigger

Feld

Bedeutung

Alert Type

Pulldown Feld zur Auswahl des Alarmtyps. Folgende Alarme sind möglich:

- Thread Status

Auslösen eines Alarms bei einem bestimmten Thread Status (up / down).

- Engine Status*

Auslösen eines Alarms bei einem bestimmten Engine Status (running / dead).

- Thread Held

Auslösen eines Alarms, wenn Nachrichten in der Outbound Queue gehalten werden.

- Transactions/sec

Auslösen eines Alarms, wenn die Anzahl der definierten Sekunden überschritten wird.

- Forward Count

Auslösen eines Alarms, sobald die definierte Anzahl von Nachrichten weitergeleitet wurden.

- Error Count

Auslösen eines Alarms, sobald eine definierte Anzahl von Verarbeitungsfehler überschritten wurde.

- Inbound Latency

Auslösen eines Alarms, wenn die eingestellte Zeit überschritten wurde (Eingang und Xlate).

- Outbound Latency

Auslösen eines Alarms, wenn die eingestellte Zeit überschritten wurde (OB Thread – Auslieferung).

- Total Latency

Auslösen eines Alarms, wenn die eingestellte Zeit überschritten wurde (Eingang – Auslieferung).

- Inbound Queue Depth

Auslösen eines Alarms, wenn die Anzahl der Nachrichten in der Inbound Queue überschritten wird.

- Outbound Queue Depth*

Auslösen eines Alarms, wenn die Anzahl der Nachrichten in der Outbound Queue überschritten wird.

- Last Receive*

Auslösen eines Alarms, wenn der Thread innerhalb der definierten Zeit keine Nachrichten empfangen hat.

- Last Send

Auslösen eines Alarms, wenn der Thread innerhalb der definierten Zeit keine Nachrichten ausgeliefert hat.

- Protocol Status*

Auslösen eines Alarms, wenn der Protokolltreiber einen bestimmten Status erhalten hat (initializing, opening, up, down, closing, error, ineof).

- System CPU %

Auslösen eines Alarms, wenn die Auslastung der CPU einen definierten Wert überschritten hat.

- Wait CPU %

Auslösen eines Alarms, wenn die CPU eine bestimmte Zeit auf Rückmeldung wartet (sleep im Tcl).

- USER CPU %

Auslösen eines Alarms, wenn der User die CPU über den definierten % Satz auslastet.

- Idle CPU %

Auslösen eines Alarms, wenn die CPU eine bestimmte Zeit im Leerlauf verbringt.

Virtual Memory % Free

Auslösen eines Alarms, wenn der freie virtuelle Hauptspeicher einen bestimmen % Satz erreicht.

Virtual Memory % Free

Auslösen eines Alarms, wenn der freie virtuelle Hauptspeicher eine bestimmte Größe in Kilobytes erreicht..

Disk I/O Per Second

Auslösen eines Alarms, wenn die Anzahl von I/O Operationen (Input / Output) pro Sekunde erreicht wird.

Disk % Full

Auslösen eines Alarms, wenn die Auslastung der Festplatte einen definierten Prozentsatz erreicht hat.

Disk Free Space

Auslösen eines Alarms, wenn der freie Festplattenplatz eine definierte Größe in Kilobyte erreicht hat.

File Change

Auslösen eines Alarms, wenn die angegebene Datei geändert wurde.

License Status

Auslösen eines Alarms, wenn die Lizenz ausläuft (in Tagen).

Mit * gekennzeichnete Alarmtypen werden häufig eingesetzt.

Delta

Auslösen eines Alarms, wenn ein definierter Alarmtyp zwischen zwei Messpunkten einen bestimmten Wert erreicht. Hier wird die Differenz zwischen der Messung A und der Folgemessung B ermittelt. Ein Delta kann nur bei numerischen Werten angewendet werden.

Source

Die Source definiert das Objekt, das beobachtet wird. In Abhängigkeit vom Alert Type kann über den Button Select… eine der folgenden Möglichkeiten gewählt werden:

- eine Datei

- ein Dateisystem

- eine Umgebungsvariable

- ein Prozess

- ein Host

- ein oder mehrere Threads

Source Count

Definition der Anzahl der Bedingungen, die erfüllt sein müssen, um diesen Alarm auszulösen.

- All

Alle Bedingungen müssen erfüllt sein.

- None

Keine Bedingung muss erfüllt sein. (Diese Option wird i. d. R. bei Umkehrung verwendet)

- At Least

Definiert eine Minimum (mindestens)

- Any

Irgendeine der Bedingungen muss erfüllt sein.

Je nach Auswahl benötigt das neben liegende Feld eine numerische Eingabe.

Comparing

Der Vergleichswert erwartet ggf. drei Eingaben.

Die Vergleichsoperationen:

< 

Messwert ist kleiner wie der Vergleichswert.

<=

Messwert ist kleiner oder gleich wie der Vergleichswert.

==

Messwert ist gleich dem Vergleichswert.

>=

Messwert ist größer oder gleich wie der Vergleichswert.

> 

Messwert ist größer wie der Vergleichswert.

!=

Messwert entspricht nicht (ungleich) dem Vergleichswert.

Differnd

Messwert ist unterschiedlich zum vorher gemessen Wert.

Between

Messwert liegt zwischen den beiden eingegebenen Werten.

In

Messwert entspricht einem der angegeben Werte.

Die verfügbaren Vergleichsoptionen sind abhängig von dem gewählten Alert Typ.

closing

Treiber schließt die Connection.

down

Treiber ist nicht aktiviert.

error

Ein Protokoll Fehler ist aufgetreten.

ineof

Das Ende der Datei wurde erreicht.

initializing

Der Treiber initialisiert sich.

opening

Es wird versucht eine Verbindung aufzubauen.

up

Die Verbindung steht.

Duration

Definition wie häufig bzw. wie lange eine Bedingung erfüllt sein muss, damit der Alarm ausgelöst wird.

Once

Der Alarm wird ausgelöst, sobald die Bedingung erfüllt wurde.

N Seconds

Der Alarm wird ausgelöst, sobald die Bedingung N Sekunden lang erfüllt ist.

N Minutes

Der Alarm wird ausgelöst, sobald die Bedingung N Minuten lang erfüllt ist.

Je nach Auswahl benötigt das neben liegende Feld eine numerische Eingabe.

Alert Host

Adresse des Remote Client, der den Alarm empfangen soll.

Time Window

Sollte der Alarm nur zu bestimmten Zeiten aktiviert sein, kann im Time Window die Zeit eingestellt werden. Werden keine Einstellung vorgenommen, so ist der Alarm 24 Stunden, 365 Tage aktiviert.

 

Action

Feld

Bedeutung

Action List

Anzeige der eingerichtet Aktionen.

Alert Action

Spezifizierung welche Aktion ausgeführt werden soll, wenn der Alarm ausgelöst wurde:

Callback

Meldung wird an eine Applikation gesendet. Hierfür ist die Eingabe von Host und Port nötig.

TCL

Eine Folge von TCL Befehlen wird ausgeführt. Der TCL Code muss direkt in das unten liegene Eingabefeld bzw. in den TCL Script Editor eingegeben. Ein Aufruf einer TCL Prozedur ist nicht möglich.

Exec

Ausführung eines Shell Befehles, z. B. Aufruf einer Batchdatei.

Notify

Gibt eine Nachricht auf dem Bildschirm aus, dass der Alarm getriggert wurde.

Button Apply

Übernimmt die Änderungen zu der aktuellen Aktion und zeigt sie in der Action List an.

Button Cancel

Abbruch der Änderung zu dieser Aktion, ohne die Änderung zu speichern.
