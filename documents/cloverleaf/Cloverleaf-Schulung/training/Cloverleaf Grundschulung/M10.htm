Lookup Table Configurator

Lookup Tables sind Konvertierungstabellen, die Werte zwischen 2 Systemen übersetzten. Diese Tabellen werden in die Übersetzungen (Xlates) eingebunden. Lookup Tabellen sind entweder unidirektional oder bidirektional. Die Bidirektionalität ist nur dann möglich, wenn eine 1:1 Zuordnung der Werte zu Grunde liegt.

Klassische Beispiele für den Einsatz von Lookup Tabellen im Bereich der Stammdaten sind:

 

- Stationsbezeichnungen

- Fallstatus

- Geschlechtskennzeichen

- Fachabteilungen

- Pflegeklassen

- Familienstand

- Zimmernummern

- Kostenstellen

- Ländercodes

- Bettenstellplätze

- Ereignisse / Events

- usw.

 

Eine weitere Einsatzmöglichkeit wäre die Einbindung einer Tabelle in eine Fallunterscheidung oder in eine Selektion.

 

Funktionsweise

·         Der Lookup Table Configurator unterteilt sich in die Fenster In und Out.

·         Durch die Aktion New Append wird eine neue Zeile eingefügt und das Wertepaar kann eingegeben werden. Weitere Zeilen können durch die TAB-Taste erzeugt werden.

·         Eine Sortierung (aufsteigend bzw. absteigend) kann durch einen Klick auf die Spaltenüberschrift In bzw. Out ausgelöst werden.
