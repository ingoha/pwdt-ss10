Troubleshooting nach einem Stromausfall

Prozesse und Threads starten nicht mehr bzw. sind noch gestartet (?)!

 

Werden im Taskmanager keine „hciengine“ Prozesse angezeigt, so sind auch keine Cloverleafprozesse aktiv.

-          Löschen Sie die Dateien „pid“ und „cmd_port“ aus den jeweiligen Prozessverzeichnissen. (%hcisitedir%\exec\processes)

Normalerweise werden diese Dateien beim Stoppen des Prozesses automatisch gelöscht, was bei einem Stromausfall jedoch nicht mehr geschehen kann.

 

Der Monitor Daemon startet nicht mehr (?)!

 

Wird im Taskmanager kein „hcimonitord.exe“ Prozess angezeigt, so ist auch kein Monitor Daemon aktiv.

-          Löschen Sie die Dateien „pid“ und „cmd_port“ aus dem Verzeichnis %hcisitedir%\exec\hcimonitored.

 

Lässt sich der Monitor Daemon noch nicht starten, könnte die Datei monitorShmemFile beschädigt sein.

-          Stellen Sie sicher dass keine Prozesse (hciengine) aktiv sind.

-          Rufen Sie ein Shell Window auf.

-          Vergewissern Sie sich, dass Sie auf der korrekten Site stehen (showroot).

-          Löschen Sie die Datei %hcisitedir%\exec\monitorShmemFile.

-          Erstellen Sie mit dem Befehl „hcimsiutil –R“ eine neue monitorShmenFile Datei.

-          Starten Sie über die Remote Commands den Monitor Daemon.

 

Die Datenbank wurde beschädigt!

 

Können Sie noch Daten aus der Datenbank retten? => Dann retten Sie diese!

 

Mit folgenden Befehlen können Sie die Datenbank initialisieren:

hcidbinit –A             Initialisiert alle Datenbanken

hcidbinit –r              Initialisiert die Recovery Datenbank.

hcidbinit –e             Initialisiert die Error Datenbank.

hcidbinit –i               Initialisiert die ICL registry Datenbank.

hcidbinit –C             Bereinigt alle Control Files.

hcidbinit –f               Unterdrückt die Nachfrage (Sind Sie sicher…)

 

Sollte sich selbst mit dem Befehl „hcidbinit –ACf“ die Datenbank nicht initialisieren lassen, so müssen die Datenbank Dateien ausgetauscht werden.

- Löschen Sie alle Dateien aus dem Verzeichnis %hcisitedir%\exec\databases

- Kopieren Sie die Dateien aus dem Verzeichnis siteProto\exec\databases in Ihre Site. Entfernen Sie das Attribut „read only“.

 

Nachgefragt

Der Client kann sich nicht connecten

 

- Häufig werden unterschiedliche Versionen zwischen Client und Server verwendet. Überprüfen Sie die Versionen.

 

- Oft ist der Patchlevel nicht identisch, d.h. der Client hat einen anderen Patch als der Server. Ziehen Sie die Patchlevel identisch.

 

- Verfügt der Server über mehrere Netzwerkkarten, so sollte im ServerAdmin die Exported Server Adresse eingetragen werden.
