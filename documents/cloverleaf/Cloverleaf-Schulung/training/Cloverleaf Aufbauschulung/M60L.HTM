1. Write an IB-TPS procedure that creates a reply message
Update the current Network Configuration used in the JKA lab:
Create a new 'cardio' thread and set its protocol to TCP/IP server, on the portnumber displayed in table 60.1L, with 4 bytes, exclusive, length encoding of the messages. Route all messages, using a raw route detail, to the archive thread. The 'cardio' system expects a reply for each message it sends. The acknowledgement messages consist of the constant string "ACK" followed by the first ten bytes of the original message.
Develop a TPS procedure called createReply that creates an acknowledgement message based on the data message it processes. Use the OVER disposition. Apply the createReply TPS procedure in the Network Configurator using the appropriate UPoC.
1.1 Test your procedure with the aid of the testing tool.
1.2 Test the Network Configuration
Run the engine and use the hcitcptest commandline tool to simulate the cardio system. Send a number of messages with a random content to Integrator and check the replies.
