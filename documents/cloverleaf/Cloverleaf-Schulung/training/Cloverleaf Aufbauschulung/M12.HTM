Module 12: New Features

This module aims to give a quick rundown of additional functionalities in the latest (and future) versions of the Integrator.

Version 5.0 will be released in September 2002, version 5.1 is announced to be released in November.

Integrator 5.0/5.1 Features List

Feature Description Benefit
Integrated IDE Unified IDE, mostly preserving the look and feel of the old GUIs. Easy of use.
XML Schema support Import of existing Schema's into the Integrator. (version 5.0 and 5.1 have no support for namespaces yet, nor for the use of non-ASCII characters in tagnames, support for these issues has been announced for the first release/revision after 5.1) Use Schema-based XML formats like any other message format in the Integrator.
TCP/IP Multi-Server The TCP/IP and PDL TCP/IP protocol drivers have been extended to support multiple TCP/IP clients connecting to the same TCP/IP port. Supports multiple (query) TCP/IP clients using a single server thread on a single port.
PDL TCP/IP Close-after-write The PDL TCP/IP protocol driver now support the Close-after-write option. This option was previously only supported by the TCP/IP driver. Allows for (query) connections that exist only during the message exchange process.

Version 3.8.1P was released in May 2002.

Cloverleaf 3.8.0/3.8.1 Features List

Feature Description Benefit
HTTP Support Communicate their data through HTTP protocol with and without encryption. Inter-enterprise application integration using HTTP and XML - getting through firewalls and implementing B2B.
Multi-version sites on CL running on Windows NT The ability to run multiple versions Integrator on the same Windows NT box. Will allow users to run multiple CL sites on one Windows NT systems
XML Enhancements and Support for industry standard XML formats Native support for XML. Integrator will not have to translate XML data into CL internal format. Support for industry standard XML formats. Support for schemas as well as recursion in XML B2B integration power and support for industry supported XML implementations.
Serial communications enhancements Better control and configurability for serial communications Better serial communications control
MQ Series Support / Enhancements General enhancements to MQSeries support in CL: 1) To support the MQ Series 5.1. 2) To access message head information in MQ Series via either Tcl extension or Integrator message metadata. 3) To be able to send and receive messages from remote queue. 4) To implement the better error handling. 5) Improve the performance of message receiving. We need to implement interruptible event-driven model for message reading instead of timer-driven polling model. Top-notch MQ-Series support for MQ-Series users.
Engine Memory Utilization Performance gains Improving Engine memory utilization. Better Engine performance and memory reuse when memory is freed.
Smart Queuing - better queue depth control Improvements made to provide better tools for queue management and dealing with exceptions. Better queue management and depth control - minimizing problems with overflowing queues.
Segment Bulk Copy Segment Bulk Copy will replicate and relocate a segment from an input message into the output message. More power less work.
Clear Field Command in Translate Clear Copy will provide a new, one-step COPY Action that will allow the user to replicate a single constant or component (e.g. "null") into the many components contained in a targeted output field, segment, or group. This will enable easy "clears" of output target fields, segments, or groups when faced with the needs indicated above. Better functionality.
ICL API for Remote control of threads Developing some Tcl extensions that have the functionality of hcicmd commands, such as thread start and stop, and message holding and forwarding etc., and be called in TPS. Better remote control of threads.
Enhanced Message Resend / Resend from Error DB Enabling a set of messages stored in a file to be inserted into the engine for processing and resending. The enhancement gives the user more control over where the messages are inserted in the processing flow. It can also be used to send a file of messages for testing.
Integrated Scheduling Through various Network Configurator dialog boxes, users may specify schedules for the timer event-related protocol drivers such as FileSet/FTP and UPOC. Event types would include a single time event, an hourly event, a daily event and a weekly event. Timer controls for CL events.  
Improve FTP Protocol (batch ftp command)/VMS Raw Mode The goal of these changes is to provide Integrator users the standard functionality they would expect if running a standalone ftp client and server. Improved FTP functionality
Java GUI's Speed/Size Improvements High-efficient Java GUIs for clients Faster client performance.
Better Support for patch versions In order to release changes for Integrator code, we would either send out updated files to particular clients, or make a complete new release to upgrade to. Neither of these are good to support from a maintenance standpoint. The professional approach is to create a patch system that will support multi-levels of patches and to quickly restore an old patch if the update does not work. More customer convenient installation paradigm for patch releases.
Java2 1.3 Support Delivers improved memory utilization and faster GUIs. Better performance and more efficient memory utilization.
TCL 8.2 (or better) Support Upgrading to the latest Tcl can boost engine performance Better engine performance, more throughput.
Add TPS Shutdown Mode Adding the Shutdown mode in addition to the existing Start and Run modes in the parameter of TPS procedure. Better usability
