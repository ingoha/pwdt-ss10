HL7 Pathnaming

Wie im Testingtool deutlich erkennbar, enthält das HL7 Format wiederholbare Elemente auf Segment- und Feldebene. Diese Elemente bzw. deren Inhalte werden mittels HL7 Pathnaming eindeutig adressiert.

Der HL7 Path beinhaltet Segmentgruppen, Segmente, Felder und deren jeweilige Wiederholungszähler. Nummerierungen beginnen immer mit der Basis 0. Somit trägt die erste Segmentgruppe die Adresse 0, die zweite Gruppe die Adresse 1, die dritte Gruppe die Adresse 2, usw.

 

Segmentgruppen

Segmentgruppen bestehen aus zwei oder mehreren Segmenten, die zusammen als eine Einheit, entweder optional oder wiederholbar, gekennzeichnet sind. Optionale Segmentgruppen werden mit den eckigen Klammer [ ] abgebildet, wiederholbare Segmentgruppen werden mit geschweiften Klammern abgebildet { }.

Wiederholungszähler

 

HL7 bietet die Möglichkeit der Wiederholungen von Segmentgruppen, Segmenten und Feldern. Aus diesem Grund wird jedem Element direkt ein Wiederholungszähler angestellt. Wiederholungszähler werden durch runde Klammern ( ) dargestellt. Die Zahl in der Klammer gibt die Wiederholung an.

0(0).MSH(0).00013

Jedes Element des HL7 Path besitzt somit einen Wiederholungszähler mit Ausnahme des letzten Elements. Hier kann der Wiederholungszähler weggelassen werden. Es wird der Defaultwert 0 für die erste Wiederholung angehängt.

 

Das erste Element des Pfades 0(0) definiert die erste Segmentgruppe, in der ersten Wiederholung.

Das zweite Element des Pfades MSH(0) definiert das Segment MSH in der ersten Wiederholung.

Das dritte Element des Pfades 00013 gibt das Feld in der ersten Wiederholung an.

Die einzelnen Elemente eines HL7 Pfades werden durch einen Punkt ( . ) von einander getrennt.

EVN

Die Segmente NK1, GT1, PV1 und BLG bilden eine neue Segmenthauptgruppe 1(0).

Innerhalb dieser Hauptgruppe sind die Segmente NK1 und GT1 zu einer optionalen Untergruppe zusammen gefasst [ ].

Für Untergruppen wird ein eigener Zähler 0(0) dem Hauptgruppenzähler angefügt. Mit dieser Patherweiterung lassen sich beliebige Verschachtelungstiefen darstellen.

 

HL7 Felder und Subfelder

 

HL7 bietet eine Vielfalt von Datentypen. Beginnend von einfachen Strings, über Datumsfelder bis hin zu Adressfeldstrukturen. Die Adressierung der Felder erfolg direkt über die Feldnummern. Besteht ein Feld aus mehreren Subfeldern, so können diese Subfelder direkt über die Feldnummer + Subfeldnummer angesprochen werden. Hierzu wird die Subfeldnummer in eckigen Klammern [ ] mit einem Punkt ( . ) als Elementtrenner an die Feldnummer angefügt.

 

Beispiel Patientenname:

 

0(0).PID(0).00108(0).[0]       Patient Name (last)

0(0).PID(0).00108(0).[1]       Patient Name (first)

 

 

Anmerkung:

 

Die genaue Adresse eines Feldes, kann mit dem Testingtool oder mit dem SMAT von CLOVERLEAF® auf einfache Weise ermittelt werden. Je nach gewählten „Detail level“ werden die Pathadressen in ihrer Tiefe angezeigt.
