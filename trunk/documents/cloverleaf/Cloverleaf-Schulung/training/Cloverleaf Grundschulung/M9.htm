XML Datensatz

Daten werden zunehmen im XML Format ausgetauscht. Um dieser Entwicklung Rechnung zu tragen, wurde der XML Package Manager im CLOVERLEAF® implementiert. Die Beschreibung der Datensatzstruktur wird vom jeweiligen Lieferant zur Verfügung gestellt und kann in CLOVERLEAF® importiert werden. CLOVERLEAF® unterstützt die zwei Definitionstypen DTD (Document Type Definition) und das XML Schema (XSD). Eine manuelle Definition des Datensatzes wie bei den Formaten FRL, VRL, HRL oder HL7 entfällt dadurch.

Die Konfigurationsdateien des XML Package Manager werden im Verzeichnis $HCISITEDIR/formats/ gespeichert.

 

XML Package Manager

Die DTD oder XSD Dateien werden mit dem Package Manager geladen und in eine CLOVERLEAF® interne Beschreibungssprache (OCM-Files) umgesetzt. Sowohl die DTD und XSD Datei, als auch die CLOVERLEAF® OCM Dateien werden im Verzeichnis $HCISITEDIR/formats/ gespeichert.

Funktionsweise

Der XML Package Manager unterteilt sich in die Fenster Client File Viewer und Server Package Viewer. Der Client File Viewer ist analog dem Windows Explorer aufgebaut und dient zum Auswählen der zu importierenden DTD oder XSD Datei. Die gewählte Datei muss in den Zwischenspeicher kopiert werden.

 

Im Server Package Viewer wird für jedes XML Format ein neuer Package/folder angelegt. Die jeweilige Datei aus dem Zwischenspeicher wird in den Package/folder eingefügt. Abschließen wird durch den Compile Vorgang die Validität der DTD oder XSD Datei geprüft und die OCM Files erzeugt.
