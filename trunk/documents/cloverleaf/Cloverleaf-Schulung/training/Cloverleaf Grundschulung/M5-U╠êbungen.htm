Grundsätzliches:

 

Alle Übungsaufgaben in dieser fünftägigen Grundschulung bauen aufeinander auf. Aus diesem Grund sollten die hier vorgegebenen Namen für Felder, Datensätze und Dateien verwendet werden.

 

Testdaten stehen im Verzeichnis %hciroot%\testdaten zur Verfügung.

 

Übungsaufgabe zum FRL Datensatz und Configurator

Ziel dieser Übungsaufgabe ist es eine Schnittstelle zwischen einem Patientenverwaltungs- und einem Küchenmanagement-System zu erstellen.

 

 

Vorgehensweise:

 

1.1 Erstellung einer Maske (Template) für die Telefonnummer und speichern als Masken.frl.

 

1.2 Erstellung des fixed-length record layouts ADM.frl. Dieser Datensatz verwendet das Template für die Telefonnummer gleich zweifach.

 

1.3 Erstellen der Datensatzstruktur für den SFP.frl

 

1.4 Testen der erzeugten fixed-length record Layouts mit dem Testingtool.

1.4 Testen der erzeugten fixed-length record Layouts mit dem Testingtool.

 

Testdaten stehen im Verzeichnis %hciroot%\testdaten zur Verfügung.

 

Das Testingtool können Sie direkt aus dem FRL Configurator über das Symbol  starten. Wenn Sie das Testingtool direkt aus einem Configurator starten, wird immer der entsprechende Notebook Tab aktiviert und mit der aktuellen Konfigurationsdatei (Testdatei) vorbelegt. Starten Sie das Testingtool aus der Launch Bar, so müssen Sie die Auswahl selber einrichten.

 

 

Ø        Geben Sie die Test Optionen an. Der ADM.frl kann mit der ADT_FRL.dat Datei getestet werden.

 

Ø        Testen Sie sowohl mit dem „Detail level 1“ und mit dem „Detail level 2“ Ihren FRL.

 

Ø        Mit dem Run Button starten Sie den Testvorgang.

Ø        Vergleichen Sie die Ausgabe der unterschiedlichen „Detail level“ miteinander. Je höher der „Detail level“ eingestellt ist, desto granularer ist die Anzeige im Testing Tool.

 

„Detail level 1“ zeigt alle Füllzeichen an. „Detail level 2“ entfernt die Füllzeichen und extrahiert die eigentlichen Daten.

 

Ø        Lassen Sie das Testing Tool offen und wechseln Sie in den FRL Configurator. Ändern Sie das Feld Satztyp des ADM Datensatzes auf eine Länge von 5 Zeichen. Speichern Sie Ihre Änderung im FRL und starteten Sie den Testvorgang erneut im Testing Tool. Die Ausgabe ist nun nicht mehr korrekt! Korrigieren Sie die Feldlänge wieder auf 4 Zeichen im FRL und speichern Sie die Änderung. Stellen Sie im Testingtool fest, ob die Änderung erfolgreich war.

 

Ø        Testen Sie nun den Datensatz SFP.frl mit den Daten der Datei SFP_FRL.dat.
