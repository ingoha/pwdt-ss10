Site Daemons

Die Site Daemons werden benötigt, um Prozesse und Threads zu kontrollieren und Befehle auszuführen. Die Prozesse im Network Monitor können erst administriert werden (Starten, Stoppen, etc.), wenn die Daemons gestartet wurden.

 

 

Lock Manager

Der Lock Manager regelt den Dateizugriff der CLOVERLEAF® Engine.
Das umfasst die Zugriffe auf

·       die Logfiles (Error Log, Prozess Log)

·       die Datenbanken (Recovery, Error)

·       die SMAT Dateien

Der Lock Manager wird nur auf UNIX Systemen benötigt.

 

Monitor Daemon

Der Monitor Daemon stellt die Monitoring Funktion des Network Monitors zur Verfügung. Erst wenn dieser Deamon gestartet ist, stehen die aktuellen Statusinformationen der Prozesse und der Threads zur Verfügung.

 

Über den Monitor Daemon wird die gewünschte Alarm-Datei geladen. Standardmäßig ist die default.alrt hinterlegt. Sobald die Alarm-Datei geladen wurde, sind die eingerichteten Alarme sofort aktiviert.
