Network Configurator - Protokolle

Nachfolgend werden die wichtigsten Protokolle vorgestellt.

 

PROTOCOL:file

 

Dieses Protokol wird i.d.R. nur in Schulungen bzw. Testumgebungen verwendet. Die Datei ist solange im permanenten Zugriff, wie der Thread gestartet ist.

 

Bereich / Element

Beschreibung

Input

Filename

 

Angabe des Pfads und des Dateinamens der Datei, die gelesen werden soll.

File Format

Auswahl des Dateiformates:

Newline-terminated

Jeder Datensatz endet mit einem Newline Zeichen (\n).

Length-Encoded

Jede Nachricht beginnt mit einem zehnstelligen Zähler, der die Gesamtlänge des Datensatzes angibt.

End-of-File Terminated

Die Datei wird bis zum Ende „am Stück“ eingelesen.

Input delay

Sollen die Nachrichten mit Verzögerung eingelesen werden, kann hier eine Eingabe in Sekunden erfolgen.

Output

Filename

 

Angabe des Pfads und des Dateinamens der Ausgabedatei.

File Format

Auswahl des Dateiformates. Analog des Input File Formats.

Append to file

Datensätze werden an eine vorhandene Datei angefügt. Andernfalls wird die Datei überschrieben.

Start-up procedures

Hinterlegung eines TCL Skripts, das beim Starten des Threads einmalig ausgeführt werden soll.

 

 

PROTOCOL:fileset-ftp

 

Cloverleaf verwendet hier das CURL FTP. Es wird ausschließlich der Passiv-Modus unterstützt. Der FTP Treiber beherrscht u. a. das Einlesen von einzelnen Datensätze aus einer Datei. Hierzu führt er einen Status mit, der nach einem Stoppen bzw. Starten des Threads an der korrekten Position, innerhalb einer Datei, weiter liest. Nach der Verarbeitung der gesamten Datei wird diese gelöscht.

Beschreibung

              Style

 

Eingabe des Einlesepfads.

eof                      Die Datei wird bis zum Ende „am Stück“ eingelesen.

              Deletion

              Scan Intervall

Anzahl der Nachrichten, die pro „Read Interval“ eingelesen werden sollen.

              Style

hl7                   Die Datei enthält mehrere HL7 Datensätze die nach dem HL7 Standard voneinander getrennt sind.

Verfügbarkeit der SSL Option

 

Die SSL (Secure Socket Layer) Option ist bei den Protokollen tcpip, pdl-tcpip, ftp verfügbar. Die benötigten Zertifikate sind nicht im Leistungsumfang des Kommunikationsservers Cloverleaf enthalten und müssen vom Kunden gestellt und verwaltet werden.
