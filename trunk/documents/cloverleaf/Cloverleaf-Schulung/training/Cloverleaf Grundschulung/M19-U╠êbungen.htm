Grundsätzliches:

 

Alle Übungsaufgaben in dieser fünftägigen Grundschulung bauen aufeinander auf. Aus diesem Grund sollten die hier vorgegebenen Namen für Threads, Xlates und Dateien verwendet werden.

 

Übungsaufgabe B zum Network Configurator

Erstellung einer neuen Konfiguration, die folgendes Szenario abbildet.

Ein Inbound Thread empfängt die Daten eines Labors für 2 Krankenhäuser. CLOVERLEAF® muss nun bestimmen welcher Datensatz an welches Krankenhaus geroutet wird.

 

 

Vorgehensweise:

 

10.1 Kopieren der TCL Dateien und Aktualisierung des Index

 

10.2 Erstellung der Xlate his_lab.xlt.

 

10.3 Einrichten der Konfiguration

 

10.4 Starten und Testen der neuen Konfiguration.

 

10.1 Kopieren der TCL Dateien und Aktualisierung des Index

Ø        Kopieren Sie die *.tcl Dateien aus dem Verzeichnis …/testdaten in das richtige Verzeichnis.

Ø        Öffnen Sie den Site Manager und aktualisieren Sie die Anzeige.

o        Sind die Prozeduren verfügbar?

10.3 Einrichten der Konfiguration

 

Ø        Erstellen Sie gemäß der Abbildung und den Hinweisen eine neue Konfiguration.

o        Ordnen Sie alle neuen Threads dem Prozess labtrixd zu.

o        Verwenden Sie jeweils das PROTOCOL: file. Aktivieren Sie die Funktion Append to file.

o        Aktivieren Sie die SMAT Funktion.

o        Aktivieren Sie Optionen Auto-start connection und Use Recovery database.

o        Zur Bestimmung der Trx Id wählen Sie bitte UPOC und hinterlegen Sie die Prozedur trxSendingFacility.

o        Richten Sie ein Acknowledgement ein, indem Sie die Prozedure gcGenBasicHL7Repy im TPS Inbound Data hinterlegen.

 

 

 

10.4 Starten und Testen der neuen Konfiguration

Ø        Starten Sie den Prozess.

Ø        Überprüfen Sie das Logfile.

o        Diskutieren Sie die Inhalte des Logfiles.

Ø        Überprüfen Sie die Statusfenster.

o        Welche Unterschiede können Sie erkennen?

o        Diskutieren Sie die Inhalte des Status.

Ø        Klicken Sie mit der rechten Maustaste auf den Pfeil (auf der Route) und schauen Sie sich die Route Properties an.

Ø        Fahren Sie die Prozesse wieder herunter.
