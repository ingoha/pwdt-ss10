Grundsätzliches:

 

Alle Übungsaufgaben in dieser fünftägigen Grundschulung bauen aufeinander auf. Aus diesem Grund sollten die hier vorgegebenen Namen für Threads, EOAliase und Dateien verwendet werden.

 

Fortsetzung (2) der Übungsaufgabe A

Ziel dieser Übung ist es den Engine Output besser kennen zu lernen und Alarme auszulösen.

 

 

Vorgehensweise:

 

10.5 Erstellung und Zuordnung des EOAlias lab_3.

 

10.6 Erweiterung der default.alrt um weitere Alarme.

 

10.7 Starten und Testen der geänderten Konfiguration.

 

10.5 Erstellung und Zuordnung des EOAlias lab_3

Ø        Erstellen Sie den EOAlias lab_3 gemäß der Spezifikation.

Enable:                     file   All submodules

Severity                    Level of 1

Speichern als:         lab_3 (Site)

Ø        Ordnen Sie den EOAlias lab_3 dem Thread his_1 fest zu.

Ø        Ordnen Sie den EOAlias enable_all dem Thread lab fest zu.

 

10.6 Erweiterung der default.alrt um weitere Alarme.

 

Setzten Sie folgende Anforderungen mit den gegebenen Möglichkeiten um:

 

Ø        Der Thread his_1 soll einen Alarm auslösen, wenn er 60 Sekunden lang keine Nachrichten erhalten hat. Als Aktion soll mit dem Befehl net send ein Text verschickt werden (z. B. net send IP Adresse „der Thread his_1 wartet seit 60 Sekunden auf Arbeit. Nun aber dallidalli“).

Ø        Wenn mindestens einer der Threads his_1 und his_2 die Dateien ausgelesen haben, soll ein Notify erscheinen.

Ø        Wenn der Prozess lab abgeschaltet wurde, soll ein Notify erscheinen.

 

10.7 Starten und Testen der neuen Konfiguration

Ø        Starten Sie den Prozess lab mit all seinen Threads.

o        Versuchen Sie das Logfile zu interpretieren.

o        Wo liegt das Logfile im Filesystem?

o        Stoppen Sie den Prozess

Ø        Starten Sie den Thread his_1.

o        Versuchen Sie das Logfile zu interpretieren.

o        Erkennen Sie die Message ID? Ist sie fortlaufend?

o        Starten Sie die anderen Threads des Prozesses lab.

o        Was können Sie erkennen?

o        Stoppen Sie den Prozess lab.

Ø        Ordnen Sie den EOAlias lab_3 dem Prozess lab zu (zuvor entfernen bei his_1).

o        Starten Sie den Prozess und verfolgen Sie die Message ID.

o        Was können Sie nun erkennen?

o        Diskutieren Sie mit der Gruppe.

o        Stoppen Sie den Prozess.

Ø        Sind die Alarme alle ausgelöst worden?
